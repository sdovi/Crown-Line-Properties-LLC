# Инструкция по экспорту статического HTML

## Шаг 1: Настройка переменных окружения (опционально)

Если вы хотите, чтобы форма отправляла заявки в Telegram, создайте файл `.env.local`:

```env
NEXT_PUBLIC_TELEGRAM_BOT_TOKEN=your_bot_token_here
NEXT_PUBLIC_TELEGRAM_CHAT_ID=your_chat_id_here
```

**Важно:** Для статического экспорта переменные должны начинаться с `NEXT_PUBLIC_`, чтобы они были доступны в браузере.

## Шаг 2: Экспорт статического сайта

Выполните команду:

```bash
npm run build
```

После выполнения команды все статические HTML файлы будут находиться в папке `out/`.

## Шаг 3: Результат

После сборки вы получите:
- `out/index.html` - главная страница
- `out/_next/static/` - статические ресурсы (CSS, JS, изображения)
- `out/logo.jpg` - логотип

## Шаг 4: Развертывание

Вы можете загрузить содержимое папки `out/` на любой хостинг:
- GitHub Pages
- Netlify
- Vercel
- Любой статический хостинг

## Примечания

1. **API Routes не работают** в статическом экспорте. Форма настроена на отправку напрямую в Telegram из браузера.

2. **Изображения:** Убедитесь, что все изображения находятся в папке `public/` перед сборкой.

3. **Telegram Bot:** Если вы используете Telegram бота, токен будет виден в коде клиента. Это нормально для публичных ботов, но убедитесь, что бот имеет ограничения по доступу.

## Структура после экспорта

